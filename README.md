# 🐕 Кронверкские псы | Avito Hack 2026

## Time-to-Sell: Глубинная аналитика ликвидности и оптимизация VAS

Проект команды **«Кронверкские псы»** в рамках хакатона Авито. Решение сфокусировано на поиске «точки перегиба» органического трафика и разработке стратегии применения платных услуг (VAS) без каннибализации естественного спроса.

---

## Бизнес-задача
Основная проблема классифайда — баланс между монетизацией и лояльностью. 
- **Слишком рано:** VAS съедает бесплатную органику (каннибализация).
- **Слишком поздно:** Объявление «зависает», продавец уходит к конкурентам (отток).

Наше решение классифицирует объявления по типам ликвидности и предлагает **динамическое окно входа для VAS** на основе статистически значимых паттернов.

---

## Методология и Технологический стек

Мы отошли от простых корреляций в пользу строгого математического аппарата:
1. **Непараметрическая статистика:** Использование U-критерия Манна-Уитни и H-критерия Краскела-Уоллиса для анализа распределений с тяжелыми хвостами (контакты в авто-сегменте).
2. **Сегментация по квантилям:** Разделение рынка на полярные группы по цене, пробегу и дельте от рыночной стоимости (IMV).
3. **Гео-анализ:** Визуализация ликвидности по регионам РФ с использованием `GeoPandas` и `russia.geojson`.
4. **Инсайт-майнинг:** Выделение специфического «Ретро-сегмента» с уникальной кривой спроса.

**Стек:** `Python`, `Pandas`, `NumPy`, `SciPy`, `Matplotlib`, `Seaborn`, `GeoPandas`, `Plotly`, `LightGBM`.

---

## Структура пайплайна

Для воспроизведения результатов ноутбуки необходимо запускать **строго последовательно**:

1.  **`1_EDA.ipynb`** — Первичный анализ, очистка данных от аномалий и ботов, восстановление пропусков.
2.  **`2_Feature_Licvid.ipynb`** — Статистическая проверка гипотез. Определение факторов, которые *действительно* влияют на скорость продажи.
3.  **`3_Segmentation.ipynb`** — Сборка финальных сегментов (Top/Worst/Middle) и формирование стратегии для алгоритмов монетизации.
4.  **`RETRO.ipynb`** — Эксклюзивное исследование рынка янгтаймеров и антиквариата. Доказательство гипотезы о «плоском спросе».

---

## Ключевые инсайты (Product Value)

* **Ретро-подписка:** Для коллекционных авто (1900–2000 гг.) суточные бусты неэффективны. Рекомендовано внедрение долгосрочного продукта — «Ретро-подписки» на 30/60 дней.
* **Оптимальное окно VAS:** Для масс-маркета («Рабочие лошадки») критическая точка — 5-7 день. Именно в этот период органический охват падает на 60%, что является идеальным моментом для пуш-уведомления о покупке VAS.
* **Гео-фильтрация:** Выявлены регионы с «перегретым» спросом, где органики достаточно для продажи любого лота в течение 3 дней — там применение VAS должно быть ограничено для сохранения лояльности.

## Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone [https://github.com/Arseniy0203/Avito_Case.git](https://github.com/Arseniy0203/Avito_Case.git)
   cd Avito_Case
   ```

2. Установите зависимости 
   ```
   pip install -r requirements.txt

   ```

3. Подготовьте данные:

* Создайте структуру папок: data/raw и data/processed.

* Поместите liquidity_base.csv и liquidity_cnt.csv в data/raw/.

* Перенесите russia.geojson.txt в data/raw/.

4. Структура репозиторий после подготовки данных


```text
├── data/
│   ├── raw/                  # Сырые данные (liquidity_base.csv, liquidity_cnt.csv, russia.geojson.txt)
│   └── processed/            # Очищенные датасеты и сгенерированные графики
├── 1_EDA.ipynb               # Очистка от шума, отсечение ботов и фейков, восстановление пропусков (IMV)
├── 2_Feature_Licvid.ipynb    # Статистическая валидация непрерывных и категориальных факторов
├── 3_Segmentation.ipynb      # Комбинаторный поиск ниш, выделение Топ/Худших сегментов, Гео-аналитика
├── RETRO.ipynb               # Специфический EDA и поиск инсайтов по янгтаймерам и антиквариату
└── requirements.txt          # Зависимости проекта
└── Кронверкские_псы.pdf      # Презентация решения
```


5. Запуск кода 

* Запускать все ноутбуки нужно последовательно для правильной работы:
    1_EDA.ipynb -> 2_Feature_Licvid.ipynb -> 3_Segmentation.ipynb -> RETRO.ipynb
